Getting Started

Welcome to Aiero — AI Agency & Technology (React + Vite single-page application). Thank you for using this template.

The project is built with React 19, Vite 8, React Router 7, and Sass. It includes multiple home demos, inner pages, blog, shop, and service layouts—routed from src/App.tsx with lazy-loaded page chunks.

Global styles load from src/styles/styles.scss (entry in src/main.tsx). Shared “inner / blog / services / shop” shells live under src/pages/layouts/.

Requirements

Install Node.js (LTS recommended, e.g. 20.x or 22.x) and use npm (included) or Yarn/pnpm if you prefer.

What's Included

After purchase on ThemeForest, download the package and extract it. Typical top-level contents:

Main folders

  • src/ — React app: App.tsx (routes), pages/, components/, data/, context/, styles/
  • public/ — Static assets served at the site root (e.g. /assets/images/, favicon.ico)
  • documentation/ — This HTML documentation (open documentation/index.html in a browser)

Installation & scripts

From the project root (where package.json lives):

  1. Install dependencies 
    npm install
  2. Development server (Vite default port 5173) 
    npm run dev

    Open http://localhost:5173

  3. Production build — output in dist/ 
    npm run build
  4. Preview production build locally 
    npm run preview

Deploying: Host the contents of dist/ on AWS Amplify Hosting or another static host such as S3/CloudFront. Configure the server to serve index.html for client-side routes (SPA fallback).

Change site title & favicon

Initial HTML title & favicon (before React mounts): edit the project root index.html — update <title>…</title> and the <link rel="icon" …> (place icons under public/, e.g. public/favicon.ico referenced as /favicon.ico).

Per-route page titles (after navigation): strings live in src/data/pages.ts (getPageMetadata() updates document.title and meta tags). Add or adjust keys for new routes.

Change logo

Logo images are loaded from public/assets/images/ (URLs like /assets/images/logo.svg). Replace files there or update the src paths in header/footer components under src/components/headers/ and src/components/footers/.

Header & navigation

Header variants and nav markup live under:

  1. src → components → headers (e.g. Header1.tsx, Header4.tsx, Nav.tsx, MobileMenu.tsx)
  2. Mobile menu structure: src/data/mobileMenu.ts

Home sections (about, hero, …)

Each home demo uses components under src/components/homes/ (e.g. homes/home-1/, homes/home-2/, …). Page entry files are in src/pages/homes/index1/ through index17/.

  1. src → components → homes → home-*

Contact section component

Reusable contact block (used in layouts and pages):

  1. src → components → contact — Contact.tsx, ContactForm.tsx, ContactBanner.tsx

Social links (header / footer)

Top-bar social icons are defined in header components (e.g. Header4.tsx). Footer socials: src/components/footers/. Use Font Awesome classes (fab fa-…); styles come from @fortawesome/fontawesome-free imported in src/main.tsx.

  1. src → components → headers or footers

Projects / portfolio

Project cards and grids use data from src/data/projects.ts and page components under src/pages/inner-pages/project/, project2/, project-single/.

  1. src → data → projects.ts
  2. src → components → others-pages (gallery / project sections as needed)

Central data & routing

Static copy, menus, blog posts, shop products, pricing tables, and page metadata are grouped under src/data/ (e.g. blog.ts, shop.ts, pages.ts, demo.ts). Routes and lazy imports are declared in src/App.tsx; the 404 page is src/pages/not-found/index.tsx (path="*").

  1. src → data
